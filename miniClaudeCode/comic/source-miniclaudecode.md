# miniClaudeCode 知识漫画

## 第1话：什么是 Claude Code？

### 场景1：开场

一个程序员小明坐在电脑前，面前堆满了代码文件，头上冒着烟。桌上放着一杯咖啡和一只胖猫。

**小明**（气泡）："这 50 万行代码，让我改一个 bug，改到猝死……"

**胖猫**（气泡）："喵~ 你需要一个编程助手！"

### 场景2：Claude Code 登场

屏幕上出现了一个蓝色的 AI 助手形象（圆润可爱风格，像哆啦A梦一样从口袋里掏出工具）。

**Claude Code**（气泡）："我来帮你！我是 Claude Code，一个住在终端里的 AI 编程助手！"

**小明**（气泡）："住在终端里？你不是聊天机器人吗？"

### 场景3：Agent Loop 概念

画面展示一个循环图：用户说话 -> AI 思考 -> 调用工具 -> 看到结果 -> 继续思考 -> ...

**Claude Code**（气泡）："我和普通聊天机器人不一样！我有一个'智能体循环'——我不只是回答问题，我还能读文件、写代码、运行命令！"

**小明**（惊讶脸）："你能自己动手干活？！"

**Claude Code**（得意脸）："对！看到 bug 就修，看到测试就跑！"

---

## 第2话：蒸馏魔法 —— 从 50 万行到 800 行

### 场景1：巨大的代码山

画面左边是一座代码堆成的大山，标注"50万行"，上面有 28 面小旗子，每面旗子写着一个子系统名字。

**小明**（仰望）："天哪，这也太复杂了……28 个子系统、100 多个工具、5 层权限……"

### 场景2：蒸馏过程

像炼金术一样的画面。一个大试管里装满了代码，通过加热（火焰上写着"分析核心"），蒸馏出精华液体滴入小瓶。

**蒸馏师**（小明变身）："蒸馏的关键：找到灵魂，去掉外壳！"

旁边标注四个保留的精华：
1. Agent Loop（灵魂）
2. 6 个核心工具（骨架）
3. 2 层权限（护甲）
4. 上下文管理（记忆）

### 场景3：蒸馏结果

画面右边是一个小巧精致的瓶子，标注"~800行"，发着光。

**小明**（自豪）："miniClaudeCode 诞生了！麻雀虽小，五脏俱全！"

**胖猫**（气泡）："喵~ 压缩比 625:1，比我压缩自己进纸盒还厉害！"

---

## 第3话：六大工具

### 场景1：工具箱打开

miniClaudeCode（小机器人形象）打开一个工具箱，里面整齐排列着 6 个工具，每个工具拟人化。

### 场景2：逐个介绍

**Bash 锤子**（肌肉男形象）："我能执行任何 shell 命令！`ls`、`git`、`npm`，随叫随到！"

**Read 放大镜**（戴眼镜的书虫）："我读文件，还带行号哦！方便你精确定位。"

**Write 钢笔**（优雅的书法家）："创建新文件？交给我！目录不存在？我帮你建！"

**Edit 剪刀手**（精确外科医生）："找到旧文本，替换成新文本。一刀切，不多不少！"

**Glob 探照灯**（侦探形象）："`**/*.py`？让我找找……全部找到！"

**Grep 猎犬**（猎犬戴墨镜）："正则表达式搜索？嗅嗅嗅……在第42行找到了！"

### 场景3：协作工作

六个工具一起工作的画面：Grep 先找到 bug 位置，Read 读取上下文，Edit 修改代码，Bash 运行测试。

**miniClaudeCode**："我们是一个团队！Agent Loop 指挥，工具执行！"

---

## 第4话：权限守卫

### 场景1：危险命令来袭

一个恶意代码怪物试图执行 `rm -rf /`。

**代码怪物**（邪恶笑）："删除一切！哈哈哈！"

### 场景2：两道防线

**第一道防线 —— 工具自检**：
BashTool 自己举起盾牌，上面写着"危险命令黑名单"。
**BashTool**："`rm -rf /`？在我的黑名单里！不准通过！"

**第二道防线 —— 权限模式**：
三扇门：
- ASK 门（警卫询问）："这个命令安全吗？请确认！"
- AUTO 门（自动门）："信任通过。"
- PLAN 门（只读门）："只能看，不能碰！"

### 场景3：安全结果

代码怪物被两道防线挡住，灰溜溜地走了。

**小明**（安心）："有了权限守卫，再也不怕误操作了！"
**miniClaudeCode**："安全第一！原版 Claude Code 有 5 层防线呢！"

---

## 第5话：动手实践

### 场景1：安装

小明在终端里输入命令，画面展示安装步骤。

```
pip install -r requirements.txt
export ANTHROPIC_API_KEY="sk-..."
python -m miniclaudecode
```

**miniClaudeCode**（从屏幕里跳出来）："我准备好了！"

### 场景2：第一次对话

小明输入："帮我看看当前目录有什么文件"

Agent Loop 开始转动（齿轮动画）：
1. 发送到 Claude API
2. Claude 返回 tool_use: glob
3. 执行 glob 工具
4. 结果返回给 Claude
5. Claude 整理输出

**miniClaudeCode**："找到了 12 个 Python 文件和 3 个测试文件！"

### 场景3：结尾

小明和 miniClaudeCode 一起工作的温馨画面。胖猫在旁边打呼噜。

**小明**："从 50 万行到 800 行，我终于理解 Claude Code 是怎么工作的了！"

**miniClaudeCode**："记住：Agent Loop 是灵魂，工具是双手，权限是护盾！"

**胖猫**（半睡半醒）："喵~ 知识就是力量……zzZ"
